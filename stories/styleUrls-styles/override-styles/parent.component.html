Overriding styles

The objective of this exercise is to understand how affects the inclusion of styles in a component.

When you put styles within component decorator, you don't insert any stylesheets in "styleUrls" array property because the compiler crashes.

On the other hand, you can push a lot of CSS files in the styleUrls property that application is how you include the path in the list.
For example, if you put two files with a same CSS rule it will be apply the last CSS rule, that is the last CSS file in the array list.

The order of application is (from general to partcular):

  • |- Styles in CSS file. if exists more than one CSS stylesheet, it apply the last rule in the last CSS file.
  • |--- Styles with a STYLE tag within html template
  • |------Inline styles
container box

All of components are isolated by default and stay into shadow DOM. if you want to modify this, you can change Encapsulation mode.
